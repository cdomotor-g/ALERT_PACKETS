ALERT / ERTS Packet Tool

Decode and encode event-reporting radio telemetry (ALERT) messages per the Bureau of Meteorology ERTS Data Formats specification (July 2003) — ALERT Binary (ABF), BCC Extended Check, Enhanced ALERT Binary (EAF) and Enhanced IFLOWS (EIF).

Decode a message

Paste a binary string — 40 bits (four 10-bit words including start/stop bits) or 32 bits (payload only). Hex like 0x07D5F8FE also works. The message is decoded against every known format; the one that passes all checks is highlighted.

Encode a message

Pick a format, enter the sensor ID and raw value(s), and get the binary message back. CRC / FCS bits are computed automatically. The bit map shows exactly where each value lands.

URL / API-style usage

This is a static GitHub Pages site, so there is no server — but you can drive the tool from the URL and get a JSON block back on the page. Add &json=1 to show the machine-readable result (and nothing runs interactively that you didn't ask for).

  • Decode: ?decode=1000001110111010101011111100001111111100
  • Decode, JSON view: ?decode=…&json=1
  • Encode: ?encode=eif&id=2784&value=1599
  • Encode with options: ?encode=bcc&id=700&hd=3&bs=12&vco=1&de=0&polarity=standard&json=1
  • Hash form also works: #decode=…
Format cheat-sheet
  • ABF — 13-bit address, 11-bit data. Words 1–2 carry check bits 10, words 3–4 carry 11.
  • BCC Extended Check — follow-up health message: 13-bit address, 5 high data bits (HD), 4 battery status bits (BS), VCO and DE error flags. Words 3–4 carry check bits 01.
  • EAF — 12-bit address, 11-bit data, battery bit B, 6 CRC bits (wind sensors substitute gust data for the CRC). Abandoned in practice.
  • EIF — 13-bit address, 11-bit data, 6-bit FCS. FCS is a CRC with generator polynomial x⁶+x⁴+x³+1 over the 24 address+data bits (address then data, MSB first).
  • All fields are transmitted least-significant bit first; each 10-bit word is start bit + 8 payload bits + stop bit.
